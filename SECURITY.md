# Security policy

Responsible disclosure
- If you discover a security vulnerability in this project (firmware, hardware design, or documentation), please report it by opening an issue labeled `security` in this repository. We use the issue tracker as the primary contact channel for security reports.
- Do NOT post exploit code or step-by-step instructions publicly.

What to include in a report
- A clear description of the issue.
- Steps to reproduce (as concise as possible).
- Impact assessment and suggested mitigations if known.
- Your contact information for follow-up.

Response process
- We will acknowledge reports within 3 business days and propose a remediation timeline.
- Critical vulnerabilities will be coordinated with upstream projects and researchers where appropriate.

Notes
- We will not publish exploit details in this repository. Patches and advisories will be posted with coordinated disclosures when available.
